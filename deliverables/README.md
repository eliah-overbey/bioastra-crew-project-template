# Deliverables

DELETE THIS INSTRUCTIONAL TEXT AFTER READING.

This directory tracks the major outputs produced during the project.

Not every project will have the same deliverables.

Create a subdirectory for each major deliverable as it is defined.

Examples:

```text
deliverables/

conference_abstract/
manuscript/
poster/
presentation/
software_tool/
visualization_dashboard/
literature_review/
grant_proposal/
```

---

# What Belongs Here?

Any substantial project output (or a link to it's location; i.e. Google docs).

Examples:

* Conference abstracts
* Manuscript drafts
* Posters
* Presentations
* Software releases
* Visualization tools
* Literature reviews
* Grant applications
* White papers

---

# Google Docs Are Acceptable

For collaborative writing projects, the actual deliverable may live outside GitHub.

Examples:

* Google Docs
* Overleaf
* Shared institutional repositories

In these cases, create a directory for the deliverable and include a README documenting the current draft location.

Example:

```text
deliverables/

conference_abstract/
    README.md
```
